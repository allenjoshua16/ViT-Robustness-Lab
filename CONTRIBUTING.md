\# Contributing to ViT Robustness Lab



Thank you for your interest in contributing to ViT Robustness Lab.



This project explores the robustness of Vision Transformers against adversarial attacks such as FGSM, PGD, CW, and DeepFool. Contributions that improve documentation, usability, experiments, evaluation, or the web interface are welcome.



\## How to Contribute



1\. Fork the repository.

2\. Create a new branch from `main`.

3\. Make your changes.

4\. Test your changes locally.

5\. Submit a pull request with a clear description.



\## Branch Naming



Use clear branch names such as:



\- `docs/update-readme`

\- `fix/frontend-upload-bug`

\- `feature/add-attack-visualization`



\## Commit Message Format



Use simple, descriptive commit messages:



\- `docs: add contributing guidelines`

\- `fix: resolve image upload issue`

\- `feature: add robustness score visualization`



\## Pull Request Guidelines



Before opening a pull request:



\- Make sure your branch is up to date with `main`.

\- Keep the pull request focused on one clear change.

\- Explain what you changed and why.

\- Add screenshots for UI changes when relevant.

\- Avoid committing secrets, API keys, virtual environments, or generated files.



\## Areas for Contribution



Useful contribution areas include:



\- Improving setup and usage documentation

\- Adding examples or screenshots

\- Improving adversarial attack explanations

\- Fixing frontend or backend bugs

\- Improving model evaluation reports

\- Adding tests or validation checks

\- Improving UI/UX for attack visualization



\## Code of Conduct



Be respectful, constructive, and clear when discussing issues, pull requests, or project decisions.
